== Архитектура ==
  Один Pod, контейнеры общаются через unix-sockets:
    pg-gvm        — PostgreSQL (только unix-socket)
    gvmd          — Vulnerability Manager (TCP 9390 + unix /run/gvmd)
    gsad          — Web API (HTTP 9392, unix-socket к gvmd)
    redis-server  — VT-кэш для ospd-openvas
    ospd-openvas  — координатор сканов

  Init-контейнеры копируют feed-data из образов Greenbone в PVC:
    gpg-data, vulnerability-tests, notus-data, scap-data,
    cert-bund-data, dfn-cert-data, data-objects, report-formats

== ⚠ Первый старт ==
  Init-контейнеры скачают ~5GB feed-данных. Это занимает 10-30 минут
  на хорошем канале. Пока init не закончится — gvmd не стартует.
  Прогресс:
    kubectl -n {{ .Release.Namespace }} get pod -l app.kubernetes.io/component=openvas
    kubectl -n {{ .Release.Namespace }} describe pod -l app.kubernetes.io/component=openvas | grep -A2 "Init Container"
