== Окно сканирования ==
  jira/confluence запускаются каждый день и берут диапазон
  [сегодня - {{ .Values.config.scanFromDaysAgo }} дней .. сегодня - {{ .Values.config.scanToDaysAgo }} дней].
  upload  — каждый час, шлёт накопленные SARIF в Hub ({{ .Values.config.hubUrl }}).
  cleanup — раз в сутки, удаляет уже залитые SARIF старше {{ .Values.config.cleanupMaxAgeDays }} дней.

== Внешние системы ==
  Hub:        {{ .Values.config.hubUrl }}
  Jira:       {{ .Values.config.jiraUrl }}
  Confluence: {{ .Values.config.confluenceUrl }}

  Перед первым прогоном проверьте, что:
  - Hub-продукты {{ .Values.config.hubJiraProjectId }} и {{ .Values.config.hubConfluenceProjectId }} существуют;
  - сервисный токен HUB_TOKEN имеет scope upload_report;
  - Jira/Confluence PAT-ы дают права читать issues/pages.

== Состояние сканера (PVC) ==
  Один общий ReadWriteOnce PVC хранит state-файлы и очередь SARIF между всеми
  четырьмя CronJob'ами:

    kubectl -n {{ .Release.Namespace }} get pvc {{ include "chart.fullname" . }}-results

  Заглянуть внутрь (debug):
    kubectl -n {{ .Release.Namespace }} run debug --rm -it --restart=Never \
      --image=busybox \
      --overrides='{"spec":{"containers":[{"name":"debug","image":"busybox","command":["sh"],"stdin":true,"tty":true,"volumeMounts":[{"mountPath":"/app/results","name":"r"}]}],"volumes":[{"name":"r","persistentVolumeClaim":{"claimName":"{{ include "chart.fullname" . }}-results"}}]}}' \
      -- sh

  ⚠ SARIF-файлы содержат сами найденные секреты в виде plain-text. PVC
  должен быть на StorageClass с шифрованием at-rest. Подробнее — в README
  чарта, раздел «Sensitivity warning».

== Ручной прогон (smoke-test) ==
  Самое безопасное начать с upload — он не зовёт Jira/Confluence, только
  скидывает в Hub то что уже накопилось:

    kubectl -n {{ .Release.Namespace }} create job --from=cronjob/{{ include "chart.fullname" . }}-upload \
      manual-upload-1
    kubectl -n {{ .Release.Namespace }} logs -f job/manual-upload-1

  Если upload ругается на отсутствие SARIF — это норма для свежей установки.
  Дальше можно потриггерить jira/confluence:

    kubectl -n {{ .Release.Namespace }} create job --from=cronjob/{{ include "chart.fullname" . }}-jira \
      manual-jira-1
    kubectl -n {{ .Release.Namespace }} logs -f job/manual-jira-1
